do_cancel(operation_id)¶
Type:virtual
Parameters:operation_id (int) – the identifier of the running operation, as returned by the function that started it

Cancel a running method.

The derived class must implement the cancel vmethod in order to honour the request correctly. Otherwise, the operation will not be interrupted.

In all cases, if this function is called on an ongoing operation, the corresponding callback will be called with the Grl.CoreError.OPERATION_CANCELLED error set, and no more action will be taken for that operation after the said callback with error has been called.

New in version 0.1.14.

Deprecated since version 0.1.16: use Grl.operation_cancel () instead.

do_may_resolve(media, key_id)¶
Type:

virtual
Parameters:
  • media (Grl.Media) – a media on which we want more metadata
  • key_id (object) – the key corresponding to a metadata we might want
Returns:

True if there’s a possibility that source resolves key_id for

missing_keys:an optional originally empty list

Return type:

(bool, missing_keys: [object])

Checks whether key_id may be resolved with source for media, so that the caller can avoid calling Grl.MetadataSource.resolve () if it can be known in advance it will fail.

If the resolution is known to be impossible because more keys are needed in media, and missing_keys is not None, it is populated with the list of Grl.KeyID that would be needed.

This function is synchronous and should not block.

set_metadata(media, keys, flags, callback, *user_data)¶
Parameters:
  • media (Grl.Media) – the Grl.Media object that we want to operate on.
  • keys ([GObject.ParamSpec] or None) – a list of Grl.KeyID whose values we want to change.
  • flags (Grl.MetadataWritingFlags) – Flags to configure specific behaviors of the operation.
  • callback (Grl.MetadataSourceSetMetadataCb) – the callback to execute when the operation is finished.
  • user_data (object) – user data set for the callback

This is the main method of the Grl.MetadataSource class. It will get the values for keys from media and store it permanently. After calling this method, future queries that return this media object shall return this new values for the selected keys.

This function is asynchronous and uses the Glib’s main loop.

New in version 0.1.4.

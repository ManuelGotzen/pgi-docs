MediaArt.get_file(artist, title, prefix, file)¶
Parameters:
  • artist (str) – the artist
  • title (str) – the title
  • prefix (str) – the prefix for cache files, for example “album”
  • file (Gio.File or None) – the file or None
Returns:

cache_file:the location to store a Gio.File pointing to the user cache path, or None
local_file:the location to store a Gio.File pointing to a cache file in the same filesystem than file, or None.

Return type:

(cache_file: Gio.File, local_file: Gio.File)

Gets the files pointing to cache files suitable for storing the media art provided by the artist, title and file arguments. cache_file will point to a location in the XDG user cache directory, meanwhile local_file will point to a cache file that resides in the same filesystem than file.

When done, both Gio.File s must be freed with GObject.Object.unref () if non-None.

New in version 0.2.0.

MediaArt.process_file(buffer, mime, type, artist, title, file)¶
Parameters:
  • buffer (bytes or None) – a buffer containing file data, or None
  • mime (str) – MIME type of buffer, or None
  • type (MediaArt.Type) – The type of media
  • artist (str) – The media file artist name, or None
  • title (str) – The media file title, or None
  • file (Gio.File) – File to be processed
Returns:

True if the file could be processed.
Return type:

bool

Processes a media file. If you have extracted any embedded media art and passed this in as buffer, the image data will be converted to the correct format and saved in the media art cache.

If buffer is None, libmediaart will search the parent directory of file for image files that are likely to be media art for file, and if one is found it will be saved in the media art cache.

If file is on a removable filesystem, the media art file will be saved in a cache on the removable file system rather than on the host machine.

New in version 0.2.0.

MediaArt.strip_invalid_entities(original)¶
Parameters:original (str) – original string
Returns:original stripped of invalid characters which must be freed. On error or if original is empty, None is returned.
Return type:str

Strip a albumname or artistname string to prepare it for calculating the media art path with it. Certain characters and charactersets will be stripped and a newly allocated string returned which you must free with GLib.free ().

This functions is used internally by MediaArt.get_file () and MediaArt.get_path (). You will not normally need to call it yourself.

This function provides the following features: 1. Invalid characters include: ()[]{}_!@#$^&*+=|\/”’?~; 2. Text inside brackets of (), {}, [] and pairs are removed. 3. Multiples of space characters are removed.

New in version 0.2.0.
